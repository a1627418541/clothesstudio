# 高级时尚编辑部风格 Web UI 重设计

**日期：** 2026-07-13
**状态：** 设计已确认
**范围：** 仅桌面 Web

## 目标

将现有 Style Studio 前端重设计为高级时尚编辑部风格，同时保留 Web
应用应有的清晰度、可访问性和运行可靠性。

本次重设计覆盖首页、诊断流程、诊断完成状态、报告页、报告组件，以及上传测试页的
视觉统一。不修改诊断 API、数据持久化、风格匹配、图片生成、身份验证或数据库行为。

## 已确认方向

- 视觉风格：高级时尚编辑部风格。
- 界面语言：仅英文。
- 信息密度：适中，易于快速浏览。
- 主要设计视口：1280 至 1440 像素的桌面 Web。
- 最小设计宽度：1024 像素。
- 本轮不包含手机端和平板端优化。

不应主动删除现有响应式类，但本轮不进行移动端专门设计或移动端视觉验收。

## 视觉系统

### 色彩

- 纸张色：柔和奶油色，用作主画布背景。
- 表面色：干净白色，用于报告纸张、表单和图片画框。
- 墨色：接近黑色，用于展示字体和主要内容。
- 次级墨色：暖灰色，用于辅助文案和元数据。
- 酒红色：深酒红，仅用于主操作、激活状态、分隔线和编辑部式强调。
- 成功、警告和错误颜色保持明确区分，但降低饱和度。

设计不得使用高光渐变、玻璃拟态、霓虹色或大型装饰色块。

### 字体

- 编辑部式展示标题使用通过现有 Next.js 字体系统加载的优雅高对比衬线字体。
- 产品文案、导航、表单控件、元数据和状态消息使用 Geist 无衬线字体。
- 展示字体仅用于品牌主张、报告标题和章节开场，不得降低表单或报告的可读性。

### 形状与层级

- 卡片使用小到中等圆角，不使用大量胶囊形容器。
- 按钮采用方正或轻微圆角。
- 大部分层级通过留白、细分隔线、色块和纸张表面建立。
- 阴影仅用于关键报告纸张、图片画框和悬浮状态表面。

### 动效

- 悬停、聚焦、上传和步骤切换使用 150 至 200 毫秒的轻微动效。
- 在 prefers-reduced-motion 下禁用非必要动效。
- 任何入场动画都不得延迟主操作的可用性。

## 页面设计

### 首页

- 使用简洁页头，品牌名称为 “Style Studio”。
- 仅保留指向真实内容或可用路由的链接。
- Hero 使用非对称布局：左侧为品牌主张和 CTA，右侧为样例报告或编辑部式视觉。
- 三步流程使用大号编辑部编号和细分隔线。
- 样例报告只展示产品当前真实提供的字段。
- Privacy 区以克制的证明要点和简洁图标呈现。
- 页面以深色编辑部式 CTA 收尾。
- 如果没有真实目标页面，移除占位的 Privacy、Terms、Pricing 和 Login 链接。

### 诊断流程

- 使用聚焦的编辑部式工作区，顶部包含紧凑品牌页头和清晰的三步进度。
- 三个上传位置采用竖向图片画框，并明确说明照片要求。
- 保留当前所有上传、重试和校验行为。
- 表单字段必须有清晰 label、单位、聚焦状态和错误消息。
- 初始化、上传、失败和分析状态必须同时通过图标、颜色和文字表达，不能只依赖颜色。
- 完成状态呈现为简洁的报告封面，并提供清晰的完整报告入口。

### 报告页

- 以报告封面式摘要开场，包含日期、用户资料和 Style Identity。
- 主推荐采用双栏编辑部跨页布局：大幅预览图配合结构化文字。
- 替代推荐使用紧凑双栏布局。
- Archetype、personality label、category 和 match score 作为编辑部式元数据呈现。
- match score 必须描述为确定性的规则匹配分数，不能描述为 AI 置信度或准确率。
- 完整造型建议按服装、发型和鞋履分组，便于快速浏览。
- 颜色方向和 Avoid 建议必须有明确视觉区分。
- 保留图片加载、失败占位和明确的手动重试行为。
- 删除不可用的 transformation image 推广区，因为它不是当前可用功能。

### 上传测试页

仅同步共享色彩、字体、表单和表面系统，不扩展页面功能。

## 组件边界

- 新增共享的 SiteHeader、BrandMark 和 EditorialLabel 组件。
- 新增可复用的基础 CSS 类，用于表面、按钮、字段、元数据和聚焦状态。
- 将首页拆分为职责清晰的区块组件，不继续保留单个超长页面组件。
- 诊断状态和 API 调用继续保留在现有诊断页面中。
- 报告数据契约继续基于 ReportRecommendation。
- 不引入新的 UI 框架或运行时依赖。

## 数据与行为

重设计必须保留以下行为：

- 匿名会话初始化；
- 三张照片上传和重试；
- 表单校验和诊断提交；
- 诊断所有权校验和加载；
- 对 PENDING 推荐自动生成预览；
- 对 FAILED 预览进行手动重试；
- archetype 元数据和旧记录的 null 回退；
- 预览图片持久化行为。

本次重设计不包含任何 API 请求或数据库结构变更。

## 错误与空状态

- 每个异步状态都必须保持稳定布局，减少内容跳动。
- 上传失败信息必须显示在对应照片位置。
- 报告图片失败时必须保留推荐文字和手动重试入口。
- 禁用操作必须在附近说明其前置条件。
- 所有可见链接和按钮必须对应真实可用行为。
- 必须清除所有乱码和异常字符。

## 可访问性

- 保留表单控件的语义 label。
- 链接、按钮、上传控件和字段必须具有可见的键盘聚焦状态。
- 状态表达不得只依赖颜色。
- 保持足够的文字对比度。
- 保留有意义的按钮和链接名称。
- 尊重 prefers-reduced-motion。

## 验证

- 运行现有完整测试套件。
- 在有价值的位置，为共享 UI 文案或渲染辅助函数增加轻量回归测试。
- 运行 ESLint。
- 运行 TypeScript no emit 检查。
- 运行 Next.js production build。
- 在浏览器中检查首页、上传步骤、资料步骤、完成状态和报告页。
- 在 1024、1280 和 1440 像素宽度下进行桌面视觉检查。
- 检查横向溢出、图片比例、聚焦状态、间距、层级、加载状态和失败状态。
- UI 验证期间不得触发真实付费图片生成请求。

## 非目标

- 手机端或平板端重设计。
- 新产品功能。
- 身份验证或账户 UI。
- Transformation image。
- API、匹配逻辑、数据库或 provider 变更。
- 新增 Privacy、Terms、Pricing 或 Login 页面。
- 暗色模式。
