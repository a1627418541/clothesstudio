## 8. Issues Found and Fixes Applied

### Issue 1: `prisma.config.ts` could not resolve `DATABASE_URL`

**Symptom:**
```
Failed to load config file ... as a TypeScript/JavaScript module.
Error: PrismaConfigEnvError: Cannot resolve environment variable: DATABASE_URL.
```

**Root cause:** Prisma 7 `prisma/config` `env()` helper expects the env variable to be available at config-evaluation time, but `.env.local` was not loaded for the Prisma CLI in this setup.

**Fix:** Changed `prisma.config.ts` to read `process.env.DATABASE_URL` directly:

```typescript
import { defineConfig } from "prisma/config";

export default defineConfig({
  schema: "./prisma/schema.prisma",
  datasource: {
    url: process.env.DATABASE_URL ?? "",
  },
});
```

---

### Issue 2: WebSocket connection error with Neon adapter

**Symptom:**
```
[TypeError: bufferUtil.mask is not a function]
Error: Connection terminated unexpectedly
```

**Root cause:** The Neon serverless driver's default WebSocket transport (`ws`) is incompatible with the Next.js dev server / bundler environment in this project.

**Fix:** Switched to Neon's HTTP fetch transport in `src/lib/prisma.ts`:

```typescript
import { neonConfig } from "@neondatabase/serverless";
neonConfig.poolQueryViaFetch = true;
```

Also removed the deprecated `neonConfig.fetchConnectionCache = true` setting after seeing the runtime deprecation warning.

---

### Issue 3: ESLint config incompatible with `eslint-config-next` 15

**Symptom:** `npm run lint` failed with module resolution errors for `eslint-config-next/core-web-vitals`.

**Root cause:** The flat-config import style used by `create-next-app` did not match the package exports of `eslint-config-next@15.5.20`.

**Fix:** Rewrote `eslint.config.mjs` using `@eslint/eslintrc` `FlatCompat` and `.next/**` ignores.

---

### Issue 4: `AnonymousSession.user` relation missing back-reference on `User`

**Symptom:** `npx prisma generate` failed with:
```
The relation field `user` on model `AnonymousSession` is missing an opposite relation field on the model `User`.
```

**Fix:** Added `anonymousSessions AnonymousSession[]` to the `User` model.
