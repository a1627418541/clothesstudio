# Sprint 3.8 Style Archetype Calibration V2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace mixed legacy/archetype recommendation generation for eligible new diagnoses with a deterministic Archetype V2 pipeline whose Top 3 recommendations, report content, and style-preview prompts all derive from one immutable validated snapshot.

**Architecture:** Keep the single existing diagnosis AI call for analysis and whole-report legacy fallback. Add an additive Prisma V2 schema, validate and score eligible Archetypes from diagnosis analysis only, enforce Primary-first macro diversity, persist three recommendations atomically, and compile V2 prompts from immutable snapshots. Keep true legacy paths available and place paid provider calls behind an exact-state CAS claim.

**Tech Stack:** Next.js 15, React 19, TypeScript 5, Prisma 7, PostgreSQL/Neon, Zod 4, Vitest 4, existing `StylePreviewImageProvider` protocol.

**Approved design source:** `docs/superpowers/specs/2026-07-14-archetype-v2-prompt-diversity-design.md`. The filename supplied in the review message does not currently exist; do not rename the committed spec without a separate request.

## Global Constraints

- Do not modify Auth, upload, payment, wardrobe, commerce, or the public image-provider protocol.
- Do not add a second AI call or recompute/backfill old reports.
- Keep `colorPalette`; never add `recommendedColors`.
- Existing records remain `LEGACY_AI` with null snapshot/version/compiler version.
- V2 report and prompt paths read only validated immutable snapshots.
- True legacy display and prompt paths stay working after every commit.
- Invalid V2 may display mirrors and an existing image, but never regenerate.
- Gender compatibility is MALE→MALE+UNISEX, FEMALE→FEMALE+UNISEX, OTHER→OTHER+UNISEX. OTHER is not a wildcard.
- Full 20-slug readiness runs after seed and before flag enablement, not per diagnosis request.
- `STYLE_ARCHETYPE_V2_ENABLED` is enabled only by exact `true`.
- Prompt safety uses structure, limits, cleanup, canonicalization, and a fixed compiler—not an expanding word blacklist.
- `matchScore` is 0–100 rule affinity, not confidence or accuracy.
- Domain failure may choose a whole-report legacy plan; infrastructure failure rolls back and uses `PERSISTENCE_FAILED` with no paid retry.
- Initial image claim expects exactly `PENDING`; explicit retry expects exactly `FAILED`.
- Every provider test injects a mock. No test contacts a paid provider.
- Do not spawn subagents without explicit user approval. Default execution is inline with `superpowers:executing-plans`.
- Each behavior task follows red test → observed failure → minimal implementation → focused green → regression green → one commit.
- Each task creates exactly one reviewable commit.
